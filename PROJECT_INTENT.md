# Agent-wiki 产品基准线

本文档是 Agent-wiki 产品方向的唯一长期基准。它只记录已经确认、跨具体工具长期成立的产品原则，不是功能清单、开发计划或实现规范。

## Agent-wiki 是什么

Agent-wiki 是一个开源的个人 AI 知识资产系统。它把用户希望长期积累的外部信息整理成用户自己拥有、可以直接阅读和编辑、也能被 AI 准确理解和使用的本地知识资产。

知识库的本质是一组 Markdown 文件。Obsidian 是当前默认的存储、阅读和管理载体，但不是永久绑定的产品前提。知识资产应能被 Agent-wiki 以外的 Agent 和项目读取、分析和使用。

Agent 是这套系统的主要使用者和调度者。Chrome 扩展、快捷指令、命令行或本地界面可以提供入口、配置和状态，但不取代 Agent 对工具的调用与编排。

## 核心处理原则

不同类型的信息源先由适合该来源的专用工具深入理解。例如，视频工具需要理解语言、画面和上下文；其他来源也应按自身特点提取信息。完成来源理解后，结果进入统一的后续处理管线，不再因为原始媒体类型形成不同的资产逻辑。

统一处理不等于丢失出处。资产仍应保留来源链接、来源类型、作者、时间、平台标识等必要信息，以便追溯原始内容。

每个成功处理的信息源都形成一份独立的来源 Markdown。它包含三种清楚区分的内容：

1. **简洁概括**：让人和 AI 快速知道这个来源主要说了什么。
2. **完整内容整理**：尽量完整地说明来源说了什么、表达了什么，保留理解原意所需的信息和上下文，但不追求逐字记录，也不按固定清单凑内容。
3. **AI 分析**：明确标识为 AI 产生，用来说明这些内容意味着什么、能做什么、有什么价值。它只依据当前来源，不读取其他知识资产，不负责外部事实核验，也不强制生成固定栏目或无实际价值的填充内容。

入库整理应忠实保留来源真正想表达的意思。可以压缩重复、寒暄、广告和明显无关内容，但不能因此改变原意、论证关系或必要上下文。来源中可能存在的错误或过时说法由后续使用这些资产的 Agent 判断，入库阶段不擅自改写为其他结论。

## 派生资产

来源 Markdown 是每次成功入库的基本产物。派生资产是可选的后续产物，不能替代、缩减或污染来源 Markdown。

派生由统一管线中的独立环节处理。具体候选格式、评分维度、阈值、日志、提示词和执行方式，应在实际调整派生功能时决定，不属于产品基准线。

候选、评分和调试记录不属于正式知识资产。只有派生资产真实生成后，才在真实存在的资产之间建立关系。

## 当前边界与未来方向

Agent-wiki 当前首先负责信息入库和知识资产整理。检索与知识库维护是后续可能发展的能力，但本文档不提前规定它们的实现方式、开发顺序或完成标准。

面向自媒体创作者的对标视频、爆款拆解、选题和创作分析不属于 Agent-wiki 的长期通用能力。这类功能以后由独立的“自媒体工作台”承载；它们仍可能使用 Agent-wiki 形成的知识资产。

早期设想中的上下文包和项目计划生成不再是产品目标。P0/P1/P2/P3、Stage、一次性任务 Spec 和历史讨论只记录当时的建设过程，不能自动成为后续开发路线。

## 文档原则

- 只有已经确认、跨工具长期成立的产品判断才写入本文档。
- 当前已经实现的能力和运行方法写入对应工具说明与技术文档。
- 当前知识资产的目录、字段和写入规则写入 `SCHEMA.md`。
- 尚未实现、尚未深入讨论或可能随开发变化的方案，不提前写成正式规范。
- 产品方向发生变化时，应先由用户确认，再修改本文档；旧文档和当前实现不能反过来覆盖产品基准线。
