# Agent-wiki 开发 AI 入口

开始分析或修改项目之前，先阅读 `PROJECT_INTENT.md`。它是产品方向的唯一长期基准。

- `SKILL.md`、`deps/*/SKILL.md` 和 `docs/` 说明当前工具的运行方式与技术事实；仍有遗留内容时，以其中的状态标记、代码和测试共同判断。
- `SCHEMA.md`、`rules/` 和 `templates/` 仍包含为兼容当前实现保留的遗留约束。只有代码和测试支持、且不与 `PROJECT_INTENT.md` 冲突的部分才是当前契约；不得据此扩展未来功能。
- 当前行为以代码和测试为准；当前实现与 `PROJECT_INTENT.md` 不一致时，应明确指出差异，不能用旧实现覆盖产品原则。
- 本地被忽略的 `obsidian-librarian/` 是废弃旧目录，不得读取、引用或用作开发依据。
- P0/P1/P2/P3、Stage、一次性任务 Spec 和历史讨论不是长期路线，不得自动解释成下一步任务。
- 未实现或尚未确认的功能不得提前写成正式规范。功能进入实际开发范围后再形成任务方案，经过实现和验证后更新对应说明。
